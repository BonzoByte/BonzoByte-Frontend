What It Is

BonzoByte in one sentence

BonzoByte is a research-driven tennis analytics platform that combines a local ASP.NET Core / C# data engine, a normalized relational database, advanced statistical modeling, neural-network inference, and an Angular frontend optimized for fast archive-based exploration.

System Architecture

Heavy computation offline, fast delivery online

.NET Core analytics engine

The core of the project is a local ASP.NET Core / C# backend that scrapes, parses, validates, and enriches tennis data before writing it into a structured relational model.

  • Data ingestion and consolidation from multiple sources
  • Parsing and validation of matches, players, tournaments, and odds
  • Generation of derived statistics, rating histories, and model inputs
  • Export of optimized JSON payloads and compressed Brotli archives

Relational source of truth

The backend uses a normalized relational database as the clean, maintainable source of truth. This keeps ingestion and feature generation strict, while archive payloads are intentionally denormalized later for fast client consumption.

  • Strict relationships and validation rules
  • Normalized schema for consistency and maintainability
  • Denormalized archive outputs for frontend speed
  • Clear separation between operational modeling and presentation data
Statistical Engine

Granular rating and performance modeling

TrueSkill at multiple resolutions

BonzoByte computes a deeply granular TrueSkill layer across match result, set result, and game result dimensions.

Surface-aware strength

Ratings are tracked not only globally, but also by surface, allowing player strength to be represented as context-sensitive rather than flat.

Rolling performance scopes

Historical metrics are aggregated across overall, weekly, monthly, and yearly scopes, creating a richer player and matchup representation.

Machine Learning

Neural-network probabilities built on engineered features

The platform includes a trained neural-network model built on a curated subset of approximately 75 selected features, chosen from a wider pool of more than 250 engineered candidate variables.

The emphasis is not only on prediction, but also on disciplined feature selection, signal quality, and evaluation against market baselines.

Archive Delivery

Why the frontend is fast

Static Brotli archives

Because the overwhelming majority of the dataset is historical and effectively immutable, BonzoByte delivers most information as static Brotli-compressed archives rather than querying a live database on every interaction.

  • Daily match archives
  • Detailed match archives with hundreds of attributes
  • Player archives and tournament archives
  • Index archives for fast search and filtering
  • TrueSkill history archives for chart rendering
  • Analytics and Model Insights payloads

Bandwidth and latency advantages

Raw JSON payloads can be dramatically larger than their Brotli-packed equivalents. That reduces transfer cost, avoids unnecessary roundtrips, and allows many interactions to happen entirely in client memory.

  • Archive-first, edge-friendly delivery model
  • Near-zero runtime dependency on the historical data store
  • Fast client-side filtering and exploration
  • Custom Brotli viewer / decompression layer inside the frontend
Frontend Application

Angular as a serious client application

Archive consumption

The Angular frontend fetches, decompresses, and presents archive content on demand, turning mostly static datasets into a responsive product experience.

Interactive research UI

Matches, players, tournaments, analytics dashboards, and model insight pages are presented through route-level pages, shared modal architecture, and rich detail views.

High-speed in-memory UX

Index archives can be loaded once and then searched or filtered instantly on the client, with no repeated backend roundtrips.

Research Layer

Prediction is only part of the story

BonzoByte compares neural probabilities against market-implied probabilities and evaluates them using probabilistic metrics such as Brier score and log loss.

It also studies blended strategies and segment-specific behavior across tours, surfaces, tournament strength, and favorite / underdog structures — all surfaced through Analytics and Model Insights.

Authentication

User platform and security

MongoDB for the user domain

MongoDB is used for account concerns such as registration, login, verification, password reset, user state, and application-level identity workflows.

Auth stack

The platform supports classic credential-based authentication as well as OAuth providers, using JWT-based session handling in the application layer.

Engineering Principles

How the project is built

  • Separation between heavy offline computation and lightweight online delivery
  • Strict dependency injection and service boundaries
  • Normalized source data with intentional denormalized read models
  • Archive-first strategy for mostly immutable datasets
  • Reusable UI patterns and shared modal architecture
  • Client-side performance through precomputed payloads
  • Incremental research-to-product workflow
  • Strong focus on maintainability, explainability, and scalability

Roadmap

What comes next

Richer feature generation

Future model iterations will incorporate additional statistical inputs such as aces, double faults, return winners, and other deeper performance signals.

Live and probabilistic extensions

Existing Markov-chain work can be connected to live match states to estimate win probability and final score paths before and during matches.

Broader platform scope

Planned work includes stronger fallback coverage, doubles support, more advanced explainability, and deeper model interpretation layers such as SHAP-style analysis.
